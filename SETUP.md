# Genesis OS Supabase Setup

## 1. Create a free Supabase project

1. Sign in at https://supabase.com and create a new project.
2. Wait for the project to finish provisioning.
3. Open the project dashboard and copy the project URL and anon public key.

## 2. Create the tasks table

In the Supabase SQL Editor, run:

```sql
create table if not exists tasks (
  id bigint generated by default as identity primary key,
  text text not null,
  done boolean not null default false,
  created_at timestamptz not null default now()
);
```

If you want to allow inserts from the public anon key, keep the table row-level security off for this demo.

## 3. Environment variables

### Codespaces

Create an untracked local env file in the workspace root so the values stay only in your Codespaces environment:

```bash
cat > .env.local <<'EOF'
VITE_SUPABASE_URL="https://your-project-ref.supabase.co"
VITE_SUPABASE_PUBLISHABLE_KEY="your-publishable-anon-key"
EOF
```

This file is ignored by Git, so your secrets will not be committed.

### Vercel

In Vercel Project Settings > Environment Variables, add:

- VITE_SUPABASE_URL
- VITE_SUPABASE_PUBLISHABLE_KEY

Make sure to redeploy after saving them.

## 4. Test the connection

1. Start the app locally with `npm run dev`.
2. Open the dashboard and add a task.
3. Confirm the task appears in the Supabase `tasks` table.
4. If the environment variables are missing, Genesis OS will continue to use localStorage.
